2โครงระบบกลาง (ทุกหน้าใช้ร่วมกัน)

assets/api.js?v=2

  • API.get/post/put/patch/del(path, body) — แนบ Authorization: Bearer <th_token> อัตโนมัติ · 401 → เด้งกลับ login
  • API.setAuth({accessToken,refreshToken,user}) เก็บใน localStorage (th_token, th_user)
  • SignalR auto-connect เมื่อมี token · event ที่หน้าจอฟัง: QueueUpdated BookingUpdated TherapistStatusChanged RoomStatusChanged DashboardSnapshot CleaningCheck — ยิง event เหล่านี้ทุกครั้งที่ข้อมูลเปลี่ยน หน้าจอจะรีโหลดเอง

5Manager · index.html (โหมดผู้จัดการ)

  • ใช้แดชบอร์ดเดียวกับ Owner แต่: ซ่อนการ์ด "รายรับเดือนนี้" + การ์ด "รายรับ-รายจ่าย/กำไรสุทธิ" แล้วแสดง "สต็อกใกล้หมด" แทน (GET/inventory → กรอง quantity ≤ minQuantity)
  • เมนูไม่มี: สิทธิ์การใช้งาน (Roles), แพ็กเกจการใช้งาน, การ์ดอัปเกรดใน sidebar
  • ที่เหลือเหมือน Owner: จัดคิว จัดกะ อนุมัติลา (หน้า staff → PATCH/therapist/{id}/leaves/{leaveId}/approve) จัดการสต็อก/บริการ
  • Backend ควรกรอง: ถ้าต้องการซ่อนกำไรจริงจาก Manager ให้ GET /dashboard ตัด field กำไร/รายเดือนเมื่อ role=Manager

  • เมนูเหลือ 2: POS (หน้าแรก) + การจอง&คิว (ดูคิวรอชำระ · โหมดดูอย่างเดียว)

8สถานะ (enum) ที่ UI ใช้

ชนิดค่า
คิว (walk-in status)0 = รอเรียก (Waiting) · 1 = กำลังให้บริการ (InService) · 2 = เสร็จสิ้น (Completed) · ยกเลิก (Cancelled)
การลา (leave)status: 0 รออนุมัติ · 1 อนุมัติ · 2 ไม่อนุมัติ — leaveType: 0 ทั้งวัน · 1 บางช่วง
สิทธิ์ใน matrix1 = เข้าถึงได้ · 2 = ดูอย่างเดียว (แบนเนอร์เหลือง + ซ่อนปุ่มแก้ไข) · 0 = ไม่มีสิทธิ์ (ซ่อนเมนู + บล็อกหน้า)
therapistSelectionMode0 = ลูกค้า/รีเซปเลือกเอง · 1 = ระบบจัดให้อัตโนมัติ
